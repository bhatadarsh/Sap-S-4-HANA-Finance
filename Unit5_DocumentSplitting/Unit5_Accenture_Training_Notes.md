## 1. Assignment 1: Document Splitting & Posting

**Objective**: Post financial documents involving vendors and GL accounts, analyze how document splitting functions between entry and general ledger views, and configure document change rules.

### Step 1: Post a Vendor Invoice (T-code `FB01`)
*   **T-Code**: `FB01` (Post Document)
*   **Action**: 
    1. Enter Document Date, Type (`KR` for Vendor Invoice), and Company Code.
    2. For the first line item, enter **Posting Key `31`** (Vendor Invoice Credit) and select the Vendor account. Enter the invoice amount.
    3. For the second line item, enter **Posting Key `40`** (Debit) and select an Expense GL account. Enter the same amount. Note: Ensure a Profit Center is assigned to the expense line if required by your field status.
    4. Simulate and **Post** the document.

### Step 2: View the Vendor Invoice & Analyze Document Splitting
*   **T-Code**: `FB03` (Display Document)
*   **Action**: 
    1. Enter the document number generated in Step 1.
    2. **Entry View**: The screen initially shows the Entry View (what the user manually keyed in): one line for the Vendor (Creditors) and one line for the Expense.
    3. **General Ledger View**: Click the **General Ledger View** button (or navigate via menu `Environment` > `Document Environment` > `General Ledger View`). Notice how the Vendor line (which inherently doesn't have a profit center assigned in master data) inherits the Profit Center/Segment from the Expense line item. This is **Document Splitting** in action, ensuring zero-balance reporting at the Profit Center level.

### Step 3: Post an Accounting Document involving 2 GL Accounts (T-code `FB01`)
*   **T-Code**: `FB01`
*   **Action**: 
    1. Enter Document Date, Type (`SA` for GL Document), and Company Code.
    2. Enter **Posting Key `40`** (Debit) and a GL account. Enter an amount and assign Profit Center A.
    3. Enter **Posting Key `50`** (Credit) and a different GL account. Enter the same amount and assign Profit Center B.
    4. Simulate and **Post**.
    5. Navigate to `FB03`, view the document, and switch to **General Ledger View**. Observe how the system handles the balancing of profit centers (potentially generating additional clearing lines if zero-balance splitting is active).

### Step 4: Configure Document Change Rules
*   **Path**: `SPRO` > `Financial Accounting` > `Financial Accounting global settings` > `Document` > `Rules for Changing Documents` > `Document Change Rules, Document Header` (or `Line Item`)
*   **Action**: 
    1. The goal is to allow the "Document Date" (`BKPF-BLDAT`) to be editable even after the document is posted.
    2. Click "New Entries". 
    3. Enter Field name: `BKPF-BLDAT` (Document Date).
    4. Check the box that says "Field can be changed". Save.

### Step 5: Edit the Posted Accounting Document
*   **T-Code**: `FB02` (Change Document) or from `FB03` by clicking the "Change" (pencil) icon.
*   **Action**: 
    1. Enter the document number of one of the documents you just posted. You can find existing documents by checking the `BKPF` table (Accounting Document Header) using T-code `SE16N`.
    2. Click on the document header icon (the hat icon).
    3. Because of the configuration in Step 4, the **Document Date** field will now be editable (white background instead of grey).
    4. Change the date and click **Save**.

## 2. Topics Covered Today (Concepts)

*   **Document Splitting**: A powerful feature in New General Ledger (and S/4HANA) that automatically splits line items in an accounting document based on specific characteristics (like Profit Center, Segment, or Business Area). Its primary purpose is to ensure that companies can draw up complete financial statements (Balance Sheet and P&L) for entities below the company code level (e.g., getting a balanced balance sheet for a specific Profit Center).
*   **Entry View vs. General Ledger View**: 
    *   **Entry View**: Shows the document exactly as it was entered by the user or interface.
    *   **General Ledger View**: Shows the document as it is posted in the Universal Journal (`ACDOCA`). This view reveals the additional lines or inherited characteristics generated by the Document Splitting functionality.
*   **Posting Keys (`31`, `40`, `50`)**:
    *   `31`: Vendor Invoice (Credit to Vendor).
    *   `40`: General Ledger Debit.
    *   `50`: General Ledger Credit.
*   **Rules for Changing Documents**: SAP follows strict auditing principles. Once an accounting document is posted, financial amounts and posting keys can NEVER be changed. However, non-financial fields (like text, reference numbers, and occasionally dates) can be configured as "changeable" using document change rules. Field `BKPF-BLDAT` corresponds to the Document Date in the header table (`BKPF`).
