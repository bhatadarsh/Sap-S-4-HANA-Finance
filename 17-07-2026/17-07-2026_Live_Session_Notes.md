## 1. Live Configuration Walkthrough

### Part A: Recurring Documents Workflow
The session detailed the exact steps to set up and execute recurring entries (like monthly rent).

1.  **Create Number Ranges (`OBA7` / `FBN1`)**: Before creating recurring documents, a specific number range interval must exist (often interval `X1` for document type `X1` - Recurring Entry Document).
2.  **Create Recurring Document (`FBD1`)**: The user creates a template document (e.g., Debiting Rent A/C and Crediting Cash A/C for `25000`). The system asks for a First Run Date, Last Run Date, and Interval (e.g., monthly). *This template does not update GL balances.*
3.  **List Recurring Documents (`F.15`)**: A report to view all active recurring document templates to verify they are set up correctly.
4.  **Execute Recurring Documents (`F.14`)**: This is the batch transaction that SAP uses to actually read the templates and post the real accounting documents for the current month.
5.  **View Posted Document (`FB03`)**: Checking the financial document that was automatically generated by the `F.14` batch run.

### Part B: G/L Account Line Item Display (`FBL3N`)
The session heavily focused on viewing and managing open items, specifically using an "Outstanding Expenses A/C" (`100500`).

*   **T-Code `FBL3N` (G/L Account Line Item Display)**: The screenshots show the selection screen where the user can filter by **Open items** (not yet paid), **Cleared items** (paid), or **All items**.
*   **Visual Status Indicators**: The session highlighted the importance of the status icons in SAP:
    *   🔴 **Red Circle**: Open Item (Not yet cleared/paid).
    *   🟢 **Green Circle/Square**: Cleared Item (Matched/Paid).
    *   🔻 **Red Down Arrow**: Overdue Open Item.
    *   ⚠️ **Yellow Triangle**: Parked Item.

### Part C: Clearing and Payments
The screenshots of the Line Item Display reveal different types of payment and clearing transactions being practiced on the Outstanding Expenses account:
*   **Standard Payment**: Paying the exact amount of an open invoice, completely clearing it (turning it green).
*   **Partial Payment**: Paying a portion of the invoice. The original invoice remains open (red), and the partial payment is posted as a separate open item (red) referencing the original.
*   **Residual Payment**: Paying a portion of the invoice, but SAP clears the original invoice (turns it green) and creates a brand new open invoice (red) for the remaining balance.

## 2. Topics Covered Today (Concepts)

*   **Open Item Management (OIM)**: A crucial setting in GL Master Data (`FS00`). If a GL account (like a bank clearing account or outstanding expense account) has OIM enabled, it means every line item posted to it must eventually be matched and cleared against an offsetting entry. This is how accountants track what is unpaid vs. paid.
*   **Recurring Entries vs. Regular Postings**: Recurring entries (`FBD1`) act purely as reference templates. They live in a separate table in SAP and do not impact the general ledger until the execution program (`F.14`) is run to generate actual postings.
*   **Partial vs. Residual Payments**: 
    *   *Partial* leaves the original open and adds a new open item. Best for audit trails when you want to see the original invoice amount unchanged.
    *   *Residual* clears the original and creates a new open item for the remainder. Best when you want your open items list to only show the exact remaining amounts owed.
